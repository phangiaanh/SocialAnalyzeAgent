# Explore More Agent — Design Spec

**Date:** 2026-06-15
**Status:** Approved (pending written-spec review)
**Component:** `SocialAnalyzeAgent` (new dedicated service) + `OpenClawModeSkills` gateway patch (trigger)

---

## 1. Summary

The Epaphras Telegram bot sends an hourly carousel of trending posts. Each card has a
`📊 Analyze` button that currently returns "coming soon". This project makes that button do
real work: an **"Explore More"** agent that, for a chosen post, runs three analyses and
delivers a Vietnamese report back into the Telegram chat:

1. **Fact-check** — extract checkable claims and label each `supported | disputed | unverifiable`.
2. **Comment attitude** — sentiment distribution, top themes, representative quotes.
3. **Predicted reaction** — likely 24–48h trajectory, risks, momentum.

The heavy work runs on a **separate dedicated runtime** (GreenNode / VNG AgentBase), exposed as
a public HTTP service. OpenClaw triggers it asynchronously and the result is delivered back
through OpenClaw's existing public URL.

### Key decisions (from brainstorming)

| Decision | Choice |
|---|---|
| Comment source | SocialCrawl comment endpoints (confirmed to exist for TikTok/Reddit/Threads) |
| Execution | Async: tap acks instantly, agent runs in background, posts result |
| Button on tap | Flip to "⏳ Đang phân tích…", change `callback_data → cb_noop` (re-tap inert) |
| Agent brain | Hybrid: Python orchestrates, LLM reasons via targeted structured calls |
| Where it runs | Dedicated runtime with a public endpoint; OpenClaw calls it |
| LLM | OpenAI-compatible **sidecar** on the runtime, default model `qwen/qwen3-5-27b`, configurable |
| Fact-check grounding | SocialCrawl cross-reference + comment signal (no new deps) |
| Result delivery | Callback into OpenClaw via its **public URL** + gateway token (send-only v1) |
| Report format | Single structured message |
| Report language | Always Vietnamese |
| Stack | Python + FastAPI |
| Future modes | Mode-agnostic core + per-mode analysis profiles + pluggable steps |

## 2. Architecture & flow

```
User taps "📊 Analyze" on a carousel card
        │
        ▼
[OpenClaw gateway patch]  (in-process, ~instant)
  1. editMessageReplyMarkup: Analyze button → "⏳ Đang phân tích…", callback_data → cb_noop
  2. POST job to SocialAnalyzeAgent public endpoint  (/analyze)
  3. answerCallbackQuery + return immediately
        │
        ▼
[SocialAnalyzeAgent · dedicated runtime · FastAPI]
  - POST /analyze → validate, register job, return 202 {job_id}
  - Background task (asyncio):
      resolve profile → fetch comments → fact-check → attitude → predict → format VI report
  - On done/error: POST result back to OpenClaw public URL (gateway token)
        │
        ▼
[OpenClaw gateway]  (via public URL send API)
  - Send the Vietnamese report as a reply to the original card (chat_id + message_id)
```

### 2.1 Trigger contract — OpenClaw → Agent (`POST /analyze`)

```jsonc
{
  "job_id": "uuid",                       // correlation id generated by OpenClaw
  "mode":  { "id": "esports", "label": "Esports", "icon": "🎯" },   // mode context (future-proof)
  "topic": { "id": "esports", "label": "Esports", "icon": "🎯" },
  "tick_id": "1781489278",
  "post": {
    "platform": "tiktok", "post_id": "...", "url": "...", "text": "...",
    "author": "@eslcounterstrike", "language": "en",
    "likes": 11552, "views": 335690, "comments": 239, "shares": 1167,
    "score": 0.0429, "age_hours": 0.0
  },
  "delivery": { "chat_id": 717110884, "message_id": 1234 },
  "callback": { "url": "https://openclaw-111735-epaphras.agentbase-runtime.aiplatform.vngcloud.vn",
                "token": "<gateway token>" }
}
```

### 2.2 Callback contract — Agent → OpenClaw

The agent calls OpenClaw's public URL (authenticated with the gateway token) to deliver a
text message to `chat_id` as a reply to `message_id`. The exact send mechanism (token-protected
HTTP route vs WS command — both are present on the gateway) is confirmed against the running
gateway during implementation. Logical payload:

```jsonc
{
  "job_id": "uuid",
  "status": "ok" | "error",
  "report_text": "🔍 Explore More — …",   // full VI report, or a short error notice
  "delivery": { "chat_id": 717110884, "message_id": 1234 }
}
```

### 2.3 Why send-only v1

The tap-time flip to "⏳ Đang phân tích…" (with `callback_data → cb_noop`) happens in-process and
fully satisfies the "processing, can't re-click" requirement — **no edit-from-callback needed**.
The report reply is the completion signal. A "✅ Đã phân tích" finalize is optional polish:
during implementation we test whether the gateway's public API exposes `editMessageReplyMarkup`
to an external token-authed caller. If yes, add the flip with no architecture change; if not,
fall back to a tiny co-located receiver in the OpenClaw pod (the proven `poll_daemon` pattern,
which already calls the Telegram Bot API directly). Not required for v1.

## 3. Service components (`SocialAnalyzeAgent`)

```
SocialAnalyzeAgent/
  app/
    main.py            # FastAPI: POST /analyze (202 + job_id), GET /health
    jobs.py            # in-memory job registry + asyncio background runner; job_id dedup
    schemas.py         # Pydantic: AnalyzeRequest, CallbackPayload, step result models
    config.py          # env: LLM model name, SocialCrawl key, OpenClaw url+token, callback secret
    pipeline.py        # orchestrator: resolve profile → run ordered steps → format → callback
    llm.py             # OpenAI-compatible client → qwen sidecar; strict-JSON call + retries
    socialcrawl.py     # comment-fetch adapters (extends the existing search client patterns)
    report.py          # assemble the Vietnamese single-message report (≤ 4096 chars)
    callback.py        # deliver result back via OpenClaw public URL + gateway token
    profiles/
      __init__.py      # registry: mode_id → AnalysisProfile; DEFAULT_PROFILE
    steps/
      base.py          # Step protocol: (ctx) -> validated result
      factcheck.py     # claim extraction + SocialCrawl cross-ref + comment signal
      attitude.py      # comment sentiment distribution + themes + quotes
      predict.py       # 24–48h reaction + risks + momentum
  tests/               # pytest, mock SocialCrawl + LLM
  Dockerfile
  requirements.txt
  README.md
```

Each step is isolated: input = structured `AnalysisContext`, output = a validated Pydantic
model. Steps never call Telegram or each other directly; the orchestrator wires them.

## 4. Pipeline (one job)

1. `POST /analyze` → validate → register job (dedup on `job_id`) → return `202 {job_id}`.
2. Background task:
   1. **Resolve profile** from `mode.id` (falls back to `DEFAULT_PROFILE`).
   2. **Fetch comments** via SocialCrawl for `post.platform` + `post.post_id`/`url`
      (up to `profile.comment_sample_size`, default ~200; paginate within budget).
      Graceful empty if the platform has no comment endpoint or none exist.
   3. **Fact-check**: LLM extracts checkable claims → for each, SocialCrawl cross-reference
      search + comment corroboration/dispute signal → label + confidence.
   4. **Attitude**: LLM classifies the comment sample → sentiment distribution, top themes,
      representative quotes.
   5. **Predict**: from post trajectory (`score`, velocity proxy, `age_hours`) + attitude +
      fact-check → 24–48h reaction, risks, momentum arrow.
   6. **Format** the Vietnamese single-message report.
3. **Callback** → OpenClaw public URL delivers the report as a reply.

### Report shape (Vietnamese, single message)

```
🔍 Explore More — {mode.label} · {Platform}
"{post.text truncated}"
————————————————————
✅ KIỂM CHỨNG  (độ tin: {low|medium|high})
• "{claim}" — {✅ xác nhận | ⚠️ tranh cãi | ❓ chưa rõ}
  {evidence summary}

💬 THÁI ĐỘ BÌNH LUẬN  ({N} mẫu)
🟢 Tích cực {x}%  🟡 Trung lập {y}%  🔴 Tiêu cực {z}%
Chủ đề nổi bật: {theme (count), …}
"{quote}" · "{quote}"

🔮 DỰ ĐOÁN PHẢN ỨNG (24–48h)
{↑/→/↓} {prediction}
Rủi ro: {risk}
🔥 đà: {rising|steady|fading}
```

## 5. Extensibility for future modes

The core is **mode-agnostic** — no hardcoded categories. Extension points:

- **Mode-aware context.** Every job carries `mode` and `topic`; prompts inject
  `profile.domain_hint` (e.g. "esports/gaming", "tài chính", "giải trí") so the same LLM steps
  adapt across domains.
- **Analysis profiles.** `profiles/` maps `mode_id → AnalysisProfile`:
  ```python
  AnalysisProfile(
      domain_hint: str,
      comment_sample_size: int = 200,
      steps: list[str] = ["factcheck", "attitude", "predict"],
      language: str = "vi",
      prompt_overrides: dict = {},
  )
  ```
  `DEFAULT_PROFILE` applies to any mode without an entry → **new modes work with zero code
  changes**; tuning one mode is adding a profile, not editing the pipeline.
- **Pluggable steps.** The pipeline runs `profile.steps` in order, resolved from a step
  registry. A new analysis dimension = add a `steps/` module + list it in a profile. Removing
  or reordering is config-only.

## 6. Error handling

- **LLM flakiness:** every LLM call uses a strict JSON schema + validation + bounded retries.
  On persistent failure, that section degrades to "không thể phân tích phần này" rather than
  failing the whole report.
- **SocialCrawl failures / no comments:** skip that signal, note it in the report
  ("không lấy được bình luận"); fact-check falls back to cross-reference plausibility only.
- **Whole-job failure / timeout:** callback with `status:"error"` → OpenClaw posts a short
  "❌ Không thể phân tích bài này, thử lại sau." (If the optional edit capability is wired,
  re-enable the Analyze button so the user can retry.)
- **Idempotency:** `job_id` dedup; duplicate triggers for an in-flight post are ignored.
- **Stale snapshot:** trigger includes `tick_id`; if OpenClaw's snapshot has rotated the result
  still delivers as a standalone reply (it references the post text, not live snapshot state).

## 7. Testing

- **Per-step unit tests** with mocked LLM + SocialCrawl (deterministic fixtures), mirroring the
  existing `OpenClawModeSkills/tests` style.
- **Report formatter golden tests** — Vietnamese output, length ≤ 4096, section structure,
  graceful-degradation variants.
- **Contract tests** — `/analyze` request and callback payload schemas (Pydantic).
- **Profile resolution tests** — known mode → its profile; unknown mode → `DEFAULT_PROFILE`;
  custom `steps` list honored.
- **Gateway-patch test** (in `OpenClawModeSkills`) — `cb_analyze` → button-flip-to-processing +
  `callback_data → cb_noop` + trigger POST is issued.

---

## 8. Out of scope (v1)

- Live web-search fact-check (deferred; grounding is SocialCrawl + comments).
- "✅ Done" / "Phân tích lại" button finalize (enhancement, pending edit-API confirmation).
- Persisting reports / history (results are delivered, not stored).
- Multi-language reports (Vietnamese only for v1).
